[ELK]讓Kibana可以看到Nginx Log系列(7)：設定Kibana Dashboard之一，設定index

Posted by     "Daniel" on Thursday, December 29, 2022

Kibana 可以看成是 Elastic Stack 的圖形化後台管理介面，使用者不需要具備資料庫知識也能進行相關操作。透過 Kibana 可以分析、搜尋、操作、視覺化 Elasticsearch 中的資料，也可透過 Kibana 管理 Elastic Stack 的相關設定。

在系列(5)中，已經將ELK三項服務啟動，所有服務已經連接完成。我們可以直接在網頁的URL輸入"<ELK所在的ip>:5601"，就可以看到Kibana的首頁(目前不需要輸入密碼)

一開始的歡迎頁，可以直接按Add integrations

再來會看到 integrations頁，但我們需要設定Dashboard，所以按左側的Menu Button

左側的Menu出現許多功能，而我們需要使用Analytic → Dashboard

按下Analytic → Dashbaord後，會出現Create index pattern

按下Create index pattern後，要在name的位置輸入自已在logstash.conf裡設定的index name，可以只輸入一開始的幾個字，會變動數字(如日期或自訂的其它代號可用*來代替)。再來要選擇Timestamp field(這裡代表正確log接到的時間，在logstash.conf有另外設定)，之後按下方的藍色Create index pattern按鈕

以上是Kibana設定index，再來下一篇要將接到的資料產生第一個Dashboard
